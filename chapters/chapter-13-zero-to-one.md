### 问题2：API Key测试失败

**可能原因**：
1. Key复制不完整（少了字符）
2. Key被禁用或过期
3. 网络连接问题

**解决**：
1. 重新复制Key（确保完整）
2. 在OpenRouter检查Key状态

## 13.11 复制即用模板

### 配置文件模板

```json
{
  "agents": {
    "list": [
      {
        "id": "welcomebot",
        "name": "欢迎机器人",
        "workspace": "/root/.openclaw/workspace-welcome",
        "model": {
          "primary": "openai/gpt-3.5-turbo"
        }
      }
    ]
  },
  "channels": {
    "feishu": {
      "enabled": true,
      "accounts": {
        "welcomebot": {
          "appId": "cli_xxxxxxxxxxxxx",
          "appSecret": "your_secret_here"
        }
      }
    }
  },
  "bindings": [
    {
      "match": {
        "channel": "feishu",
        "accountId": "welcomebot"
      },
      "agentId": "welcomebot"
    }
  ]
}
```

**使用方法**：
1. 复制上面的代码
2. 保存为 `openclaw.json`
3. 修改App ID和Secret
4. 放到OpenClaw配置目录
