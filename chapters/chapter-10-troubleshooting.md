## 10.1 安装问题自查清单

### ❌ 问题1：安装包下载失败

**症状**：
- 下载进度卡住不动
- 提示"网络错误"
- 下载速度极慢（<10KB/s）

**可能原因**：
1. 网络连接问题
2. GitHub访问不稳定
3. 浏览器下载限制

**解决方案**：

**方案A：检查网络**
```bash
# 测试网络连通性
ping github.com

# 如果ping不通，检查网络设置
```

**方案B：使用镜像下载**
```
如果GitHub访问慢，可以尝试：
1. 使用Gitee镜像（如果有）
2. 使用代理/VPN
3. 换个时间段下载（避开高峰）
```

**方案C：命令行下载**
```bash
# 使用wget或curl下载
wget https://github.com/xieyz1980/openclaw/releases/download/v0.1.0/openclaw-windows-x64.exe

# 或使用curl
curl -L -o openclaw.exe https://github.com/.../openclaw-windows-x64.exe
```

### ❌ 问题2：安装时提示"未知发布者"

**症状**：
- Windows提示"无法验证发布者"
- 显示红色警告框
- 无法继续安装

**原因**：软件未经过微软签名认证

**解决方案**：

**Windows 10/11**：
1. 点击"更多信息"
2. 点击"仍要运行"
3. 完成安装

**截图示例**（此处应有截图：点击更多信息按钮）

---

**Mac系统**：
1. 打开"系统偏好设置"→"安全性与隐私"
2. 点击"仍要打开"
3. 输入管理员密码

**截图示例**（此处应有截图：Mac安全性设置）

---

### ❌ 问题3：安装后打不开

**症状**：
- 双击图标没反应
- 闪退（窗口一闪就消失）
- 提示缺少DLL文件

**可能原因**：
1. 系统缺少运行库
2. 安装不完整
3. 杀毒软件拦截

**解决方案**：

**方案A：安装运行库**

Windows缺少运行库时：
1. 下载Visual C++ Redistributable
   - 访问：https://aka.ms/vs/17/release/vc_redist.x64.exe
2. 安装后重启电脑
3. 再次尝试打开OpenClaw

**截图示例**（此处应有截图：运行库安装界面）

---

**方案B：关闭杀毒软件**

某些杀毒软件会误拦截：
1. 暂时关闭360、电脑管家等
2. 将OpenClaw添加到白名单
3. 重新打开软件

**截图示例**（此处应有截图：杀毒软件白名单设置）

---

**方案C：重新安装**

如果安装不完整：
1. 卸载当前版本
2. 重新下载安装包
3. 右键"以管理员身份运行"

**截图示例**（此处应有截图：管理员身份运行）

---

### ❌ 问题4：Mac提示"已损坏，无法打开"

**症状**：
- 打开时提示"已损坏"
- 建议移到废纸篓

**原因**：Mac的安全机制阻止未认证软件

**解决方案**：

**方法1（推荐）**：
1. 打开"终端"
2. 输入命令：
```bash
xattr -d com.apple.quarantine /Applications/OpenClaw.app
```
3. 回车执行
4. 再次打开软件

**截图示例**（此处应有截图：终端执行命令）

---

**方法2**：
1. 打开"系统偏好设置"→"安全性与隐私"
2. 在"通用"选项卡中，点击"仍要打开"
3. 输入管理员密码

**截图示例**（此处应有截图：安全性与隐私设置）

## 10.2 API连接失败怎么办

### ❌ 问题1：提示"API Key无效"

**症状**：
- 配置API Key后测试失败
- 提示"Invalid API Key"
- 无法连接到AI模型

**可能原因**：
1. API Key输入错误
2. Key被禁用或过期
3. 复制时带了空格

**解决方案**：

**步骤1：检查Key格式**
```
✅ 正确：sk-or-v1-xxxxxxxxxxxxxxxx
❌ 错误： sk-or-v1-xxxxxxxxxxxxxxxx（前面有空格）
❌ 错误：sk-or-v1-xxxxxxxxxxxxxxxx（后面有空格）
```

**步骤2：重新复制Key**
1. 登录OpenRouter控制台
2. 找到Key，点击复制
3. 粘贴到OpenClaw时选择"纯文本"

**截图示例**（此处应有截图：复制API Key）

---

**步骤3：验证Key有效性**

在命令行测试：
```bash
curl https://openrouter.ai/api/v1/auth/key \
  -H "Authorization: Bearer 你的API_KEY"
```

如果返回错误，说明Key有问题。

---

### ❌ 问题2：提示"Rate Limit Exceeded"

**症状**：
- 发送消息后无响应
- 日志显示"Rate Limit"
- 一段时间后又恢复正常

**原因**：API调用太频繁，触发限流

**解决方案**：

**方案A：降低请求频率**
- 两次请求间隔至少1秒
- 避免连续快速发送多条消息
- 批量处理，减少请求次数

**方案B：升级API套餐**
1. 登录OpenRouter控制台
2. 查看当前限速
3. 升级到更高配额

**截图示例**（此处应有截图：OpenRouter配额查看）

---

**方案C：使用多个Key轮换**
```json
{
  "api_keys": [
    "sk-xxx1",
    "sk-xxx2",
    "sk-xxx3"
  ],
  "rotation": "round_robin"
}
```

---

### ❌ 问题3：提示"Model Not Available"

**症状**：
- 选择某个模型后无法使用
- 提示模型不可用
- 其他模型正常

**可能原因**：
1. 模型服务暂时故障
2. 该模型已下线
3. 账号无权限使用

**解决方案**：

**方案A：切换到其他模型**
```
如果GPT-4不可用 → 切换到Claude-3
如果Claude不可用 → 切换到DeepSeek
```

**截图示例**（此处应有截图：切换模型界面）

---

**方案B：检查模型状态**

访问OpenRouter状态页面：
- https://openrouter.ai/docs#models

查看模型是否标记为"Active"

---

**方案C：等待恢复**

如果是服务故障：
1. 查看官方状态页
2. 等待10-30分钟
3. 通常会自动恢复

---

### ❌ 问题4：网络连接超时

**症状**：
- 发送消息后长时间无响应
- 提示"Timeout"或"连接超时"
- 其他网络功能正常

**可能原因**：
1. 网络不稳定
2. API服务器响应慢
3. 本地防火墙阻止

**解决方案**：

**方案A：检查网络**
```bash
# 测试网络延迟
ping openrouter.ai

# 正常应该<100ms
# 如果>500ms或丢包严重，检查网络
```

**方案B：调整超时时间**

在配置中增加超时时间：
```json
{
  "timeout": {
    "request": 60,    // 60秒
    "connect": 10     // 10秒
  }
}
```

**截图示例**（此处应有截图：超时时间配置）

---

**方案C：使用代理**

如果网络质量差：
1. 配置HTTP代理
2. 或使用VPN
3. 切换到更稳定的网络

## 10.3 AI回复质量差怎么优化

### ❌ 问题1：回复太啰嗦

**症状**：
- 回答很长但重点不突出
- 重复说同样的话
- 用户需要反复追问

**优化方案**：

**方案A：在Prompt中明确要求**

```markdown
❌ 差的Prompt：
请介绍一下AI。

✅ 好的Prompt：
用3句话介绍AI：
1. 定义（20字内）
2. 核心应用（列举3个）
3. 未来趋势（一句话）
```

**方案B：设置输出长度限制**

```markdown
要求：
- 回答不超过100字
- 分点说明，每点一行
- 不要举例说明
```

---

### ❌ 问题2：回复不相关

**症状**：
- AI回答与问题无关
- 理解错用户意图
- 答非所问

**优化方案**：

**方案A：提供更多上下文**

```markdown
❌ 差的Prompt：
分析一下。

✅ 好的Prompt：
背景：我是做电商的，主营服装
问题：最近转化率下降了
需求：分析可能原因，给出3个改进建议
```

**方案B：使用Few-shot示例**

```markdown
参考以下格式回答：

Q: 如何提高工作效率？
A: 1. 使用番茄工作法 2. 减少会议 3. 批量处理邮件

Q: [用户问题]
A:
```

---

### ❌ 问题3：回复太笼统

**症状**：
- 回答都是"正确的废话"
- 缺乏具体建议
- 无法落地执行

**优化方案**：

**方案A：要求具体数据和案例**

```markdown
要求：
- 每个建议都要有具体步骤
- 给出可量化的目标
- 提供一个真实案例
```

**方案B：指定输出格式**

```markdown
按以下格式输出：

## 问题诊断
[具体问题描述]

## 解决方案
1. [方案名称]
   - 执行步骤：...
   - 预期效果：...
   - 所需资源：...

2. [方案名称]
   ...

## 风险提示
[需要注意的事项]
```

---

### ❌ 问题4：风格不一致

**症状**：
- 每次回复风格不同
- 有时正式有时随意
- 不符合品牌调性

**优化方案**：

**方案A：在SOUL.md中定义风格**

```markdown
## 写作风格
- 语气：专业但不失亲切
- 句式：短句为主，避免长难句
- 用词：准确简洁，避免网络流行语
- 结构：总分总，每段一个观点

## 示例
✅ 好的表达：
"AI技术正在重塑内容创作流程，建议从三个维度入手：工具选型、团队协作、质量把控。"

❌ 避免表达：
"AI真的太牛了！它超级厉害！改变了好多东西！"
```

**方案B：使用风格模板**

创建一个风格模板文件，每次写作时引用：
```markdown
[加载风格模板：专业科技文风]

要求：
- 使用第三人称
- 多用数据支撑
- 避免主观情绪词
- 段落不超过3句话
```

## 10.5 紧急恢复方案

### 🚨 情况1：完全无法使用

**恢复步骤**：

**步骤1：重启Gateway**
```bash
openclaw gateway restart
```

**步骤2：检查配置**
```bash
openclaw doctor
```

**步骤3：重置配置（谨慎）**
```bash
# 备份原配置
mv ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak

# 重新初始化
openclaw init
```

---

### 🚨 情况2：数据丢失

**恢复步骤**：

**步骤1：检查备份**
```bash
ls ~/.openclaw/backups/
```

**步骤2：恢复备份**
```bash
openclaw restore --backup 2026-03-06
```

**步骤3：手动恢复**
如果自动备份不可用，从Git或其他备份恢复。

---

### 🚨 情况3：账号被封

**可能原因**：
- API违规使用
- 账号异常活动
- 欠费未缴

**解决方案**：
1. 联系API提供商客服
2. 说明情况，申请解封
3. 准备备用账号

## 10.6 本章小结

### 📚 核心知识点

1. **安装问题**：
   - 下载失败 → 检查网络/使用镜像
   - 未知发布者 → 点击"仍要运行"
   - 打不开 → 安装运行库/关闭杀毒

3. **质量优化**：
   - 太啰嗦 → 明确要求简洁
   - 不相关 → 提供更多上下文

4. **求助技巧**：
   - 先查文档/日志/历史Issue
   - 提问时提供环境、现象、已尝试方案
   - 选择合适的求助渠道
