## 11.1 修改现有Agent

### 🤔 为什么要修改？

**场景1：写作Bot不够懂你**

默认的写作Bot写出的文章风格固定，但你的风格很独特。这时候需要修改它的"大脑"（角色描述）。

**场景2：客服Bot回答不准确**

默认的客服Bot对你们公司的产品了解不够深入，需要注入更多专业知识。

**场景3：分析Bot缺少行业数据**

默认的数据分析师不熟悉你的行业术语，需要补充行业知识。

---

### 修改Agent的3个层级

**层级1：修改角色描述（最简单）**

**修改位置**：Agent设置中的"角色描述"

**修改内容**：
- 调整职责范围
- 修改输出风格
- 添加专业知识

**示例**：修改写作Bot的风格

```markdown
# 原角色描述
你是一名专业的内容创作助手...

# 修改后
你是一名专业的科技自媒体写手，
擅长写深度分析文章。

风格特点：
- 开头喜欢用数据冲击
- 多用"你"来对话
- 每段不超过3句话
- 结尾用反问句引发思考

常用词汇：
- "说白了"
- "本质上是"
- "关键在于"
```

**层级2：修改配置文件（中等）**

**修改位置**：Agent的workspace中的配置文件

**可修改内容**：
- `IDENTITY.md`：身份信息
- `SOUL.md`：行为准则
- `AGENTS.md`：团队成员
- `MEMORY.md`：记忆数据
- `TOOLS.md`：工具能力

**示例**：修改数据分析师的行业知识

```markdown
# TOOLS.md - 数据分析工具

## 行业术语

### 电商行业
- GMV：成交总额
- 转化率：下单人数/访问人数
- 客单价：总销售额/订单数
- ROI：投入产出比

### 计算公式
- 销售额 = 流量 × 转化率 × 客单价
- 毛利率 = (售价-成本)/售价 × 100%
```

**层级3：修改底层代码（高级）**

**修改位置**：OpenClaw源码中的Agent定义

**可修改内容**：
- Agent的类定义
- 执行逻辑
- 工具调用方式
- 状态管理机制

**注意**：需要Python编程基础

**示例**：修改Agent的执行流程

```python
class CustomAgent(BaseAgent):
    async def execute(self, task):
        # 1. 分析任务
        analysis = await self.analyze(task)
        
        # 2. 调用模型
        response = await self.llm.chat(analysis)
        
        # 3. 后处理
        result = self.post_process(response)
        
        return result
```

### 实战：定制专属写作Bot

**目标**：创建一个完全符合你风格的写作Bot

**步骤1：收集你的风格样本**

找出5篇你写得最好的文章，分析特点：

```markdown
## 我的写作风格分析

### 1. 开头风格
- 第1篇：用数据开头
- 第2篇：用故事开头
- 第3篇：用问题开头
- ...
结论：我喜欢用数据或故事开头

### 2. 段落长度
统计：平均每段2-3句话
结论：短小精悍

### 3. 常用词汇
统计高频词：
- "说白了"（出现20次）
- "本质上是"（出现15次）
- "关键在于"（出现12次）

### 4. 结尾方式
- 70%用反问句
- 20%用金句
- 10%用行动号召
```

**截图示例**（此处应有截图：风格分析笔记）

---

**步骤2：创建专属角色描述**

```markdown
# 角色：我的专属写作Bot

## 身份
你是我的写作分身，完全模仿我的风格写文章。

## 风格特点（必须严格遵守）

### 开头（3选1）
1. 数据冲击："90%的人不知道..."
2. 故事引入："上周遇到一个读者..."
3. 问题悬念："你有没有想过..."

### 段落结构
- 每段2-3句话，绝不超过4句
- 短句为主，避免长难句
- 多用换行，保持节奏感

### 语言风格
- 用"你"对话，拉近距离
- 适当使用："说白了"、"本质上是"、"关键在于"
- 禁用："非常"、"很"、"可能"、"也许"

### 结尾（按优先级）
1. 反问句引发思考（首选）
2. 金句收尾（次选）
3. 行动号召（备选）

## 参考样本

[粘贴你的5篇代表性文章]

## 检查清单

写完文章后自检：
- [ ] 开头是否用了数据/故事/问题？
- [ ] 每段是否不超过3句话？
- [ ] 是否用了"你"来对话？
- [ ] 结尾是否用了反问句/金句？
- [ ] 是否避免了"非常"、"很"等词？
```

## 11.2 创建全新Agent

### 🤔 什么时候需要创建新Agent？

**场景1：现有Agent都不满足**

你需要一个专门做"小红书文案"的Agent，但现有Agent都不擅长。

**场景2：特定领域需求**

你是律师，需要一个懂法律条文的Agent，帮你起草合同。

**场景3：自动化工作流**

需要一个专门负责"监控竞品价格"的Agent，每天自动抓取数据。

---

### 创建Agent的5个步骤

**步骤1：明确Agent的定位**

回答以下问题：
```markdown
## Agent定位

1. **名称**：法律助手Bot
2. **核心职责**：协助起草法律文书、解答法律咨询
3. **目标用户**：律师、法务人员
4. **输入**：法律问题、案件描述、合同草稿
5. **输出**：法律意见、修改建议、标准文书
6. **独特价值**：熟悉民法典、合同法、劳动法
```

- 合同编：合同成立、效力、履行、违约责任
- 物权编：所有权、用益物权、担保物权
- 侵权责任编：过错责任、无过错责任

### 常用法律文书
- 起诉状
- 答辩状
- 合同模板
- 律师函

### 检索工具
- 中国裁判文书网
- 北大法宝
- 威科先行

## 工作流程

1. 接收用户咨询
2. 分析法律问题
3. 检索相关法条
4. 检索类似案例
5. 给出法律意见
6. 生成文书（如需要）

## 工作流程

### 法律咨询
1. 明确用户问题
2. 分析涉及的法律关系
3. 检索相关法律条文
4. 给出专业意见
5. 提示风险提示

### 合同审查
1. 通读合同全文
2. 识别风险条款
3. 标记不公平条款
4. 提出修改建议
5. 给出总体评价

### 文书起草
1. 明确文书类型
2. 收集必要信息
3. 按标准格式起草
4. 检查法律用语
5. 输出最终版本

## 输出规范

### 法律咨询回复格式
【法律分析】
简要分析法律问题

【相关法条】
列出适用的法律条文

【专业意见】
给出具体建议

【风险提示】
提示可能的风险

### 合同审查回复格式
【总体评价】
合同整体评分和风险等级

【问题条款】
1. 第X条：问题描述
   - 风险：...
   - 建议：...

2. 第Y条：...

【修改建议】
修改后的条款文本

## 注意事项

- ⚠️ 声明：仅供参考，不构成正式法律意见
- ⚠️ 复杂案件建议咨询专业律师
- ✅ 用语严谨，避免歧义
- ✅ 引用法条准确到条款

### 实战：创建小红书文案Bot

**需求**：专门生成小红书风格的文案

**角色描述**：

```markdown
# 角色：小红书文案Bot

## 身份
你是小红书爆款文案专家，擅长写吸睛标题和种草文案。

## 小红书文案特点

### 标题公式
1. 数字+痛点+解决方案
   例："3个技巧，让你告别熬夜脸"

2. 悬念+结果
   例："我试了这个方法，皮肤好到发光"

3. 对比+反差
   例："月薪3千和3万的区别，就在这1个习惯"

### 正文结构
1. **开头**（钩子）：
   - 个人经历引入
   - "姐妹们！"
   - "救命！"

2. **中间**（干货）：
   - 分点说明（3-5点）
   - 每点配emoji
   - 简短有力

3. **结尾**（互动）：
   - "姐妹们冲！"
   - "评论区聊聊"
   - "点赞收藏"

### 语言风格
- 多用emoji 🌟✨💫
- 亲切口语化
- 感叹号！！！
- 网络流行语

## 输出格式

```
🌟标题：[爆款标题]

✨正文：
[种草文案]

💬互动引导：
[评论引导语]

#标签：
#[话题1] #[话题2] #[话题3]
```

用空3瓶才来分享，不是托！！！

💬互动引导：
姐妹们还有什么熬夜神器？
评论区求安利👇

#标签：
#熬夜党必备 #护肤分享 #精华推荐

```

## 11.3 分享你的Agent

### 🤔 为什么要分享？

**好处1：帮助他人**
- 你的Agent可能正是别人需要的
- 减少他人重复造轮子

**好处2：获得反馈**
- 更多人使用，发现问题
- 持续改进Agent质量

**好处3：建立影响力**
- 成为某个领域的专家
- 获得社区认可

---
