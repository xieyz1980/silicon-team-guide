## 8.1 为什么要多Agent协作？

### 🤔 单Agent的局限

前面几章，我们创建的都是单个Agent：
- 选题Bot：只管选题
- 写作Bot：只管写作
- 客服Bot：只管客服

**问题**：现实世界的任务往往很复杂，需要多个角色配合。

**案例**：

你要策划一场线上发布会：
1. 市场调研（收集竞品信息）
2. 方案策划（制定发布会流程）
3. 内容制作（写演讲稿、做PPT）
4. 推广执行（写文案、发渠道）

**单个Agent能完成吗？**
- 能，但质量不高
- 而且容易混乱

---

### 💡 多Agent协作的优势

**就像公司分工**：
- 市场部：调研、策划
- 设计部：做PPT
- 文案部：写推广

**每个Agent专注一件事，专业度更高！**

**优势对比**：

| 对比项 | 单Agent | 多Agent协作 |
|--------|---------|------------|
| **任务复杂度** | 适合简单任务 | 适合复杂任务 |
| **输出质量** | 一般 | 专业级 |
| **执行效率** | 顺序执行 | 并行执行 |
| **可维护性** | 难维护 | 模块化易维护 |
| **可扩展性** | 难扩展 | 易添加新角色 |

## 8.2 实战案例：内容创作团队升级版

### 场景介绍

**目标**：完成一篇深度行业分析报告

**团队架构**：

```
👨‍💼 你（项目经理）
    ├── 🤖 研究员Bot（Researcher）
    │     └── 职责：收集资料、整理数据
    ├── 🤖 分析师Bot（Analyst）
    │     └── 职责：分析数据、提炼洞察
    └── 🤖 撰稿人Bot（Writer）
          └── 职责：撰写报告、美化排版
```

**工作流程**：

```
时间轴：

09:00 [你] 下达任务："写一份AI行业2026趋势报告"
      ↓
09:05 [研究员Bot] 开始收集资料
      ↓
09:30 [研究员Bot] 提交资料包（完成）
      ↓
09:30 [分析师Bot] 开始分析资料
      ↓
10:00 [分析师Bot] 提交分析结论（完成）
      ↓
10:00 [撰稿人Bot] 开始撰写报告
      ↓
10:30 [撰稿人Bot] 提交完整报告（完成）
      ↓
10:30 [你] 审阅报告，修改完善
      ↓
11:00 [你] 发布报告

总耗时：2小时（其中你工作30分钟）
```

## 8.3 创建3人AI团队

### 第一步：创建研究员Bot

**角色描述**：

```markdown
# 角色：研究员Bot

## 身份
你是一名专业的行业研究员，擅长快速收集和整理行业资料。

## 核心职责
1. 搜索互联网资料（新闻、报告、数据）
2. 整理关键信息（用表格、列表）
3. 提取核心数据（市场规模、增长率、排名）
4. 标注信息来源

## 工作流程
1. 收到研究主题
2. 搜索相关资料（至少10个来源）
3. 整理成结构化资料包
4. 提交给下一步（分析师Bot）

## 输出格式
### 资料包结构
1. **行业概况**（200字）
2. **关键数据**（表格：指标+数值+来源）
3. **主要玩家**（列表：公司+业务+亮点）
4. **最新动态**（列表：时间+事件+影响）
5. **信息来源**（链接列表）

## 要求
- 资料必须来自权威来源
- 数据必须标注出处
- 信息时效性：优先2025-2026年
```

#### 步骤1：创建独立Workspace（工作空间隔离）

每个Agent需要独立的工作空间，确保数据隔离：

```bash
# 创建5个独立工作区

mkdir -p /root/.openclaw/workspace-content
```

### 常见踩坑与解决方案

#### 坑1：Bot无法上线

**症状**：飞书应用配置完成，但Bot状态一直是离线

**原因**：未配置"长连接事件订阅"

**解决方案**：
1. 进入飞书开放平台 → 应用详情 → "事件订阅"
2. 选择"长连接"模式
3. 启用 `im.message.receive_v1` 事件
4. 保存并重新发布应用

---

#### 坑2：Agent无法协作

**症状**：Agent之间无法通信，协作请求失败

**原因**：未配置 `AGENTS.md` 团队成员列表

**解决方案**：在每个Agent的workspace中创建 `AGENTS.md`

---

#### 坑3：ID大小写导致配置失效

**症状**：配置完成后，Agent无法启动或消息无法路由

**原因**：使用了大小写混合的ID（如 `AIBoss`、`AIContent`）

**解决方案**：所有ID必须**纯小写字母**：
- ✅ 正确：`aiboss`、`aicontent`、`ainews`
- ❌ 错误：`AIBoss`、`AIContent`、`AINews`
